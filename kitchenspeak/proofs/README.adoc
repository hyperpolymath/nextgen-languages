= KitchenSpeak Proofs
:toc:
:icons: font

Proof-assistant lowerings of the worked examples in
`kitchenspeak/examples/`.

The target proof assistant is Agda (per
`kitchenspeak/decisions/0001-proof-assistant.adoc`). The directory
layout leaves room for a future Lean 4 port.

== Layout

[cols="1,3"]
|===
| Path | Contents

| `agda/`
| Agda 2.6.x + stdlib 2.0+ proofs. Current target.

| `lean/` _(future)_
| Lean 4 ports, after the Agda proofs stabilise. Not present at v1.0.
|===

== Current proofs

[cols="1,1,2"]
|===
| File | Corresponding example | What's proved

| `agda/Dough.agda`
| `examples/` (dough is specified in `SPEC.adoc`; no `.ks` yet)
| Termination, safety (on_fail totality), echo-witness extraction for
  the dough-kneading program. Linear + Dyadic + Choreographic + Echo.

| `agda/PoachedEgg.agda`
| `examples/poached-egg.ks`
| Termination, safety (`on_fail ABORT` / `on_fail RECOVER` totality),
  echo-witness extraction for both sync blocks, *and* the first
  Tropical refinement obligation: the GENTLE controller never crosses
  the boil envelope. Linear + Tropical + Echo.
|===

Per `../ROADMAP.adoc` Phase 1c, `Emulsion.agda` (Tropical + Linear) and
`Sear.agda` (Tropical + Echo + Primitive) join this list next, completing
the Dough / Emulsion / Sear core named in `SPEC.adoc` §6.

== Running the proofs

A build harness now lives alongside the proofs. Install Agda 2.6.4+ with
stdlib 2.0+ (registered in `~/.agda/libraries`), then:

[source,bash]
----
make -C proofs              # type-check the whole core library
make -C proofs poachedegg   # type-check PoachedEgg.agda only
make -C proofs dough        # type-check Dough.agda only
----

The `kitchenspeak.agda-lib` file declares the `standard-library`
dependency and treats warnings as errors. A future ADR may add Agda to the
development environment (Guix `guix.scm`) so CI runs `make -C proofs` on
every push (ROADMAP Phase 1a).

[NOTE]
====
The proofs in this directory have not been run in the session that
authored them — Agda is not installed in the authoring environment.
They are written against Agda 2.6.x + stdlib 2.0+ conventions and
verified by careful reading. When Agda becomes available to the
class, running the type-checker should be the first exercise.
====

== Why each proof files earns its keep

=== `Dough.agda`

Exercises the Linear + Dyadic + Choreographic + Echo quadrant. Shows
how the three production-mandatory constructs (`max_duration`,
`on_fail`, `proving @w`) lower to, respectively:

* structural recursion on a fuel parameter (termination),
* exhaustive pattern match (safety / `on_fail ABORT`),
* an existential `Σ`-type bundling the classifier's time-stamped
  event (echo witness).

Linearity is enforced by inspection of signatures, with a note
explaining the discipline; a QTT-Agda port would elevate this to a
compile-time check without changing the proof shape.
